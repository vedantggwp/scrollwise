# Scrollwise docs

- **[STATUS.md](STATUS.md)** — What’s implemented, what works, tech stack, key files. Start here for current state.
- **[BUGS.md](BUGS.md)** — Known bugs (reader only cover, feed empty, etc.) with repro and where to look.
- **[NEXT_STEPS.md](NEXT_STEPS.md)** — Prioritized plan and work silos for feature-by-feature or step-by-step work in a new chat.
- **[DESIGN.md](DESIGN.md)** — Design philosophy, principles, design language, and design-handoff specs for UI.

For a new session: open STATUS, BUGS, and NEXT_STEPS; pick one silo (e.g. “Fix feed empty” or “Fix reader only cover”) and work it through before moving on.
